# 时间记录网站开发计划（MVP优先，手机浏览器）

## 1. 目标与约束

- 目标：做一个“按时间段记录做了什么”的个人网站，核心场景是手机浏览器。
- 架构：前后端分离，前端 Vue3 + JavaScript，后端 Golang + Gin，数据库 PostgreSQL。
- 约束：不使用 TypeScript；不使用 Nginx 一类额外 Web 服务器；系统仅你个人使用。
- 登录：必须有登录能力用于防止他人使用；登录状态保持 1 个月；不做多用户数据隔离。

## 2. MVP优先实施策略

先交付“能稳定使用的最小闭环”，再迭代增强统计：

1. 登录与会话保持
2. 树形分类/事项管理
3. 目标时长与进度展示
4. 计时与手动记录（含跨零点拆分）
5. 记录编辑删除
6. 基础统计
7. 高级统计（第二阶段）

## 3. 技术方案

### 3.1 前端（Vue3 + JS）

- 移动端优先 UI（大点击区、底部主操作区、单手友好）。
- 单页应用：主要页面为“事项树”“记录”“统计”“设置/登录”。
- 本地仅保存轻量 UI 状态（如折叠态、当前计时会话恢复信息）。
- 与后端通过 REST API 通信。

### 3.2 后端（Go + Gin）

- 单体服务同时提供 API 与静态资源托管能力（不依赖 Nginx）。
- 统一鉴权中间件，保护除登录接口外的业务接口。
- 关键领域模块：认证、节点管理、记录管理、统计聚合、计时状态恢复。

### 3.3 数据库（PostgreSQL）

- 持久化：分类/事项树、时间记录、设置项、登录会话。
- 用事务保证“移动/排序/批量更新”一致性。
- 为统计常用维度建立索引（item_id、start_at、end_at、date_bucket）。

## 4. 核心数据模型

### 4.1 节点表（分类/事项）

- `id`
- `type`：`category | item`
- `name`
- `parent_id`
- `order_no`
- `hidden`
- `collapsed`
- `daily_target_minutes`（事项）
- `created_at / updated_at`

### 4.2 时间记录表

- `id`
- `item_id`
- `start_at`
- `end_at`
- `pause_duration_ms`
- `description`
- `source`：`timer | manual`
- `created_at / updated_at`

### 4.3 登录与会话

- `users`：单用户账号（用户名、密码哈希、状态）
- `sessions`：会话 token、过期时间（默认 30 天）

### 4.4 设置表

- `confirm_before_save_timer_record`（结束计时是否弹窗确认）

### 4.5 运行中计时状态

- 服务端持久化当前计时会话（事项、开始时间、暂停累计、是否暂停、暂停开始点），用于“关闭后恢复”。

## 5. 功能阶段计划

### 阶段 A：项目骨架与认证（MVP-1）

1. 初始化前后端工程与基础路由。
2. 建立 PostgreSQL schema 与迁移脚本。
3. 实现登录接口、密码校验、会话签发与 30 天有效期。
4. 前端接入登录页与受保护路由。

### 阶段 B：树形节点管理（MVP-2）

1. 分类/事项新增、重命名、删除、隐藏。
2. 子分类/事项缩进层级展示。
3. 分类折叠/展开并记忆状态。
4. 节点移动与同级排序。

### 阶段 C：目标与进度（MVP-3）

1. 为事项设置每日目标时长。
2. 展示当日已完成时长。
3. 展示进度条与百分比：进度条上限 100%，百分比可超过 100%。
4. 显示是否完成与完成程度（高可读性标识）。

### 阶段 D：计时流程（MVP-4）

1. 开始计时、暂停、继续、结束。
2. 结束后生成记录草稿并弹窗：保存或放弃，可填描述。
3. 支持关闭弹窗确认后“结束即直接保存”。
4. 计时状态持久化，浏览器关闭后再次打开可恢复。

### 阶段 E：手动记录与跨零点拆分（MVP-5）

1. 手动添加记录，支持选择日期和事项。
2. 记录跨 00:00 自动拆分为前后两条（跨多日则按天连续拆分）。
3. 拆分后保证总时长与暂停时长分摊口径一致。

### 阶段 F：记录维护（MVP-6）

1. 记录列表查看与筛选。
2. 编辑：起止时间、暂停时长、事项、描述。
3. 删除记录。
4. 编辑或删除后实时刷新进度与统计。

### 阶段 G：统计中心（MVP-7 + 增强）

MVP基础统计：

1. 总耗时、总计时次数。
2. 分类/事项耗时汇总。
3. 每日耗时趋势。
4. 时段分布（一天中各时间段做了什么）。

增强统计（第二阶段）：

1. 分类下子分类/事项占比。
2. 达标率、超额比例、未达标事项分布。
3. 时间利用结构（高频事项、连续专注段、碎片化程度）。
4. 多视角交叉统计（日期 + 分类 + 时段 + 计时次数）。

## 6. 关键规则与口径

- 进度百分比：`completed / target * 100`；进度条显示 `min(percentage, 100)`。
- 暂停时长必须 `>= 0` 且 `<= end_at - start_at`。
- 当日完成时长按“记录与自然日重叠区间”计算。
- 跨零点拆分以 `00:00:00` 为边界切分，描述与事项继承。
- 计时次数统计：每生成一条 `source=timer` 记录计 1 次。

## 7. API 与页面规划

- API 分组：`/auth`、`/nodes`、`/records`、`/timer`、`/stats`、`/settings`。
- 页面：
  - 登录页
  - 主页（树形 + 计时 + 今日进度）
  - 记录页（手动新增、编辑、删除）
  - 统计页（基础 + 增强）
  - 设置页（结束计时是否弹窗）

## 8. 验收标准（MVP）

- 可登录访问，未登录无法使用业务页面。
- 登录状态 30 天内有效。
- 树形节点管理完整且折叠态可记忆。
- 事项目标、完成时长、进度条、百分比展示正确。
- 计时可暂停/继续/结束，重开浏览器可恢复运行状态。
- 手动记录可添加，跨零点拆分正确。
- 记录可编辑删除，统计与进度实时同步更新。

## 9. 风险与应对

- 移动端复杂交互：优先按钮式移动/排序，拖拽作为增强功能。
- 时间边界错误：统一时间计算模块 + 跨日用例测试。
- 统计性能：先做 SQL 聚合 + 必要索引，后续再做缓存优化。
- 会话安全：密码哈希存储、会话过期校验、登出失效机制。
